# Hermes Project Rules — YangYuS8/blog

这是用户的个人博客仓库，站点地址是 https://blog.yangyus8.top/ 。

## 技术栈

- Astro / Fuwari
- pnpm
- 文章目录：src/content/posts/
- 每篇文章通常是一个目录，目录内有 index.md
- 使用 frontmatter 管理标题、发布日期、标签、分类、草稿状态等信息

## 写新文章流程

创建新文章时优先使用：

pnpm new-post "文章标题"

该命令会在 src/content/posts/ 下创建新目录和 index.md。

不要随便手写 urlSlug，除非用户明确要求。默认交给 scripts/new-post.js 生成。

## 文章 frontmatter 规范

文章 frontmatter 至少包含：

title
urlSlug
published
description
image
tags
category
draft
lang

常用 lang: zh_CN
draft 通常为 false，除非用户明确要求草稿。

## 写作风格

用户的博客偏个人技术实践记录，适合采用：

1. 问题背景
2. 环境信息
3. 排查过程
4. 根因分析
5. 解决方案
6. 复盘和经验总结

文章语气可以自然，不要太像官方文档。
技术细节要准确，命令要可执行。
不要编造没有发生过的操作记录。
如果缺少信息，先标注“这里需要用户补充”，不要假装已经知道。

## 常用命令

安装依赖：
pnpm install

创建文章：
pnpm new-post "标题"

格式化：
pnpm format

检查：
pnpm check
pnpm lint
pnpm build

## 提交规范

在修改前先看：
git status

修改后必须看：
git diff

构建通过后再建议提交：
git add .
git commit -m "docs: add xxx post"

除非用户明确要求，不要直接 push 到 main。
优先让用户 review diff，再由用户确认是否 push。

## RTK

如果系统安装了 rtk，优先使用：
rtk git status
rtk git diff
rtk read <file>
rtk grep <pattern> .
